# 舰队象棋 (Fleet Chess)

中国象棋变体 — 13×14棋盘，舰队主题，远程打击+延迟结算+隐藏信息。

## 游戏简介

将中国象棋改造为现代海战模拟。七种舰船单位替换传统棋子，加入巡洋舰远程轰炸（延迟一回合结算、目标对敌保密）、驱逐舰升变、潜艇免疫远程火力等机制。支持五种游戏模式与 .qp 棋谱系统。

## 棋子规则

| 棋子 | 简称 | 参考 | 数量 | 要点 |
|------|------|------|------|------|
| 旗舰 | 旗 | 帅/将 | 1 | 指挥区内正交1格；对脸主动方判负 |
| 护卫舰 | 护 | 士/仕 | 2 | 斜1格，区内或邻己方棋子可出区；可拦截射击线 |
| 巡洋舰 | 巡 | 炮/砲 | 3 | 直线1~2格；炮架规则（线上须有1颗棋子作炮架，炮架及其后方为有效目标）；锁定攻击双方公开可见红色四角框，敌回合末必定结算；护卫舰可拦截/潜艇免疫；开火后当回合不能移动 |
| 驱逐舰 | 驱 | 兵/卒 | 7 | 前进/左右1格；底线升变巡洋舰 |
| 战机编队 | 机 | 马 | 2 | 马走日，敌方别腿；不能攻击潜艇 |
| 潜艇 | 潜 | 新增 | 2 | 直线1~2格可穿行，吃子仅1格；免疫轰炸/战机 |
| 预警机队 | 警 | 相/象 | 2 | 斜向1~3格可越子，不过河；全灭则无法远程打击 |

指挥区：x∈[4,8]，红方y∈[0,4]，蓝方y∈[9,13]。海域分界位于y=6与y=7之间。

## 五种游戏模式

1. **AI对战** — 与大模型对弈，可选执红/执蓝
2. **自行推演** — 一人操控红蓝双方
3. **线下对战** — 面对面同屏，蓝方棋子反向显示
4. **对局再现** — 读取.qp棋谱回放；上一/下一回合；自动播放（4s间隔，2x/4x倍速）
5. **观AI战** — AI自对弈，读取上局棋谱要求变换策略

## 核心系统

- **开屏与引导**：启动展示 1maginations Studio 开屏（宋体）→ 初次使用进入引导页（配置API/推演步数并可连通性测试）→ 主界面纯模式选择；所有设置可随时在「设置」页修改
- **多API支持**：内置豆包/DeepSeek/谷歌/Grok/ChatGPT/LongCat/Kimi/MiMo/智谱九家服务商下拉选择（带品牌图标），模型名可填写（默认 deepseek-v4-flash），支持连通性快速测试；设置持久化保存
- **先手判定**：开局d2投掷随机决定，或在设置中指定（随机/红/蓝），无弹窗打断
- **棋谱(.qp)**：每局自动记录到文档目录 `FleetChess/qp/`，格式：`1.红，驱（2，5）->（2，6）空`、`2.蓝，巡（6，11）x（6，7）驱`（`->`移动，`x`轰炸）；每轮轰炸结算结果以 `# 结算：` 注释行写入棋谱供AI参考；点击底部日志面板可弹出完整棋谱查看
- **悔棋**：每局每方4次配额；AI对战中悔棋自动回退2半步耗1次；生死斗开关开启后全模式禁悔
- **AI协议**：将规则+完整棋谱传给大模型，按推演步数（默认4，可调1~10）分析后回复棋谱指令；`悔x`指令悔棋；违规指令自动反馈重试；Debug模式在对局下方文字框实时显示大模型完整返回（点击放大）
- **轰炸标识**：锁定攻击目标后棋盘出现红色四角框，双方公开可见，下回合敌方结束时必定结算
- **巡洋舰操作**：点击选中（移动模式）→ 再点自身切换攻击模式 → 再点自身切回
- **规则说明**：选中棋子时显示该棋子完整规则（可在设置关闭）
- **战损展示**：被吃棋子半透明散布于棋盘两侧
- **棋子动画**：280ms 平滑移动动画

## 技术栈

- Flutter 3.44.6 / Dart 3.12，亮色浅蓝主题 (#92CDDB)
- 支持9家大模型API（豆包/DeepSeek/谷歌/Grok/ChatGPT/LongCat/Kimi/MiMo/智谱），设置持久化（shared_preferences）
- 平台：Windows (.exe) / Android (.apk) / Web (localhost:11450)

## 快速开始

```bash
cd app
flutter pub get
flutter test                      # 19项规则引擎测试
flutter run -d windows            # 桌面运行
flutter build apk --release       # 构建APK
flutter build windows --release   # 构建EXE
```

API Key 在应用首页输入（仅存内存）。

## 项目结构
```
app/lib/
  models.dart            # 棋子/棋盘/状态/快照/设置
  engine.dart            # 走法生成+行动验证+轰炸结算+悔棋
  notation.dart          # .qp棋谱：记录/解析/文件IO
  config.dart            # API Key内存管理
  ai/ai_client.dart      # 大模型客户端（棋谱协议）
  ui/home_screen.dart    # 主菜单+设置
  ui/game_screen.dart    # 四种实时模式对局
  ui/replay_screen.dart  # 对局再现
  ui/board_widget.dart   # 棋盘渲染与交互
app/test/engine_test.dart # 单元测试
```

## 版本历史

- **try4** — 轰炸机制大修（目标公开红色四角框+必定结算）；棋盘改交叉点制（13×14线，棋子落线交点）；回合号半步递增（1红2蓝…）；全局设置悬浮球（可拖动，改动实时生效）；规则说明区常驻占位（棋盘固定不变形）；战损棋子原大小不透明；开屏延长并渐变过渡
- **try3** — UI大改：浅蓝棋盘+白色圆形棋子（宋体单字）+移动动画+黑虚线九宫+战损两侧展示；开屏引导+Splash+设置页；九家API提供商+模型名+连通性测试+设置持久化；Debug实时输出+规则说明+棋谱弹窗；巡洋舰至多越1颗任意棋子；Web支持+11450本地服务
- **try2** — 棋谱系统(.qp)、五种游戏模式、悔棋、d2先手、预警机改斜向1~3格、AI棋谱协议（推演步数/debug/悔棋指令/违规重试）、快速重开
- **try1** — 初始版本：规则引擎+基础UI+AI对战
